# BDD Feature Files

This folder contains Gherkin/Cucumber feature files for BDD testing.

## Files Generated
9 feature file(s) were generated from the sprint features and scenarios.

## Usage

### With Cucumber.js
```bash
npm install --save-dev @cucumber/cucumber
npx cucumber-js tests/features
```

### With Jest + Cucumber
```bash
npm install --save-dev jest-cucumber
```

### With Playwright + Cucumber
```bash
npm install --save-dev @playwright/test @cucumber/cucumber
```

## Feature File Structure
- Each .feature file corresponds to a feature in the sprint
- Scenarios include feature_id and scenario_id tags for traceability
- BDD steps follow Given/When/Then format

## Tags
- `@feature_id:<id>` - Links to the feature in ProjexLight
- `@epic_id:<id>` - Links to the epic in ProjexLight
- `@scenario_id:<id>` - Links to the scenario in ProjexLight
- `@scenario_type:<type>` - UI, API, or Integration
- `@ui_test` / `@api_test` - Test type classification

## Integration with MCP Server
The MCP server can report test results back to ProjexLight using the feature_id and scenario_id tags.
